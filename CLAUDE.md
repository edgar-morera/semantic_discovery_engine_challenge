## Commands

All commands run inside Docker. The `make` targets proxy into the `php` container via `docker compose exec php`.

```bash
# First-time setup
make init            # copy .env, build images, install deps, migrate

# Daily workflow
make up              # start all services
make restart         # recreate containers (required after .env changes)

# Tests
make test            # all PHPUnit suites
make test-unit       # unit suite only
# Run a single test file
docker compose exec php php vendor/bin/phpunit tests/Unit/Product/Domain/Model/ProductTest.php

# Code quality (all tools)
make analyse         # cs + stan + deptrac + phpmd

# Individual tools
make cs-fix          # auto-fix code style
make stan            # PHPStan level 8
make deptrac         # verify layer dependency rules

# Load sample data
make seed-products   # inserts 357 products and generates their embeddings (~minutes)
```

## Architecture

Hexagonal Architecture + DDD + CQRS. The domain has zero framework dependencies.

### Layer rules (enforced by Deptrac)
- **Domain** → no internal dependencies
- **Application** → Domain only
- **Infrastructure** → Domain + Application

### Symfony Messenger buses
Two independent buses configured in [web/config/packages/messenger.yaml](web/config/packages/messenger.yaml):
- `command.bus` — wraps handlers in a Doctrine transaction via `doctrine_transaction` middleware
- `query.bus` — no middleware

`IndexProductCommand` is the only async message: it routes to a Redis Stream (`messenger_index_product`). The `semantic_worker` Docker service consumes this queue. All other commands and queries are handled synchronously.

### Service wiring (ports → adapters)
| Domain port | Infrastructure adapter |
|---|---|
| `EmbeddingService` | `CachedEmbeddingService` → `HuggingFaceEmbeddingService` |
| `ProductRepository` | `DoctrineProductRepository` |
| `ProductSearchPort` | `QdrantProductSearchRepository` |
| `ProductIdGenerator` | `SymfonyUuidProductIdGenerator` |

`CachedEmbeddingService` is a decorator that caches HuggingFace results in Redis (TTL 5 min, key `embedding_<md5>`). Wiring lives in [web/config/services.yaml](web/config/services.yaml); Domain classes are excluded from autowiring.

### Search flow
1. `GET /products/search?q=` → `SearchProductsController` dispatches `SearchProductsQuery` on `query.bus`
2. `SearchProductsQueryHandler` generates an embedding for the query text via `EmbeddingService` (Redis-cached)
3. `ProductSearchPort::search()` queries Qdrant by cosine similarity, returns `SearchResult[]` with product IDs and scores
4. Handler fetches full `Product` entities from MySQL by ID and builds the response

### Indexing flow
1. `POST /products/{id}/index` → `IndexProductController` dispatches `IndexProductCommand` on `command.bus` → **202 Accepted**
2. Redis Stream worker picks up the command asynchronously
3. `IndexProductCommandHandler` loads the product, calls `EmbeddingService::generate()`, and passes the `Embedding` VO directly to `ProductSearchPort::index()` — the aggregate is not mutated

## Domain model

```
Product (Aggregate Root)
├── ProductId                     — UUID v4 (custom Doctrine type)
├── ProductName                   — non-empty string, max 255 chars (custom Doctrine type)
└── ProductSemanticDescription    — non-empty string (custom Doctrine type)

Embedding (Value Object, not persisted in MySQL)
└── values: float[384]            — used by EmbeddingService and ProductSearchPort only

SearchResult (Value Object, not persisted)
├── productId: ProductId
└── score: float                  — cosine similarity 0.0–1.0
```

`Embedding` is a vector search concern: it is generated by `EmbeddingService` and consumed by `ProductSearchPort`. It is never owned by the `Product` aggregate.

Embedding model: `ibm-granite/granite-embedding-97m-multilingual-r2` (384 dimensions) via HuggingFace Inference API.

## Code conventions

- `declare(strict_types=1)` in every PHP file.
- Domain exceptions only — no generic PHP exceptions.
- Tests: `tests/Unit/`, `tests/Integration/`, `tests/Functional/`. Naming: `<TestedClass>Test.php`, test methods in `snake_case`.
- API endpoints documented with OpenAPI (NelmioApiDocBundle); UI at `GET /api/doc`.

## Environment

Required env vars (see `.env.example`):
- `HUGGINGFACE_API_KEY` — HuggingFace Inference API token (free tier works)
- `QDRANT_DSN`, `REDIS_DSN`, `DATABASE_URL` — set automatically by Docker Compose defaults
